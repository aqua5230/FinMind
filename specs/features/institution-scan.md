# Feature Spec: 法人籌碼掃描

更新日期：2026-04-16

## 目標

建立正式的法人籌碼掃描功能。

目前程式已有雛形，但產品狀態仍是待做。

這份 spec 的目標是把功能邊界寫清楚，避免 AI 因為看到程式碼存在就誤判已完成。

## 規格狀態

草稿。

待使用者確認後，才可開始實作。

## 使用者需求

使用者希望找出近期有法人籌碼支撐的台股。

初步方向：

- 外資連續買超
- 投信加碼
- 外資與投信要分開看，不要混成單一法人指標
- 可能結合月營收 YoY
- 可能限制小型股或流動性

此功能是研究與掃描工具。

不是實盤買進訊號。

## 目前已有雛形

後端：

- `stock_report/api/institution_scan.py`
- `/api/institution-scan`
- 使用 FinMind `TaiwanStockInstitutionalInvestorsBuySell`

前端：

- `frontend/app/page.tsx` 有 `institution` tab
- `frontend/lib/api.ts` 有 `fetchInstitutionScan()`

但這些不代表功能完成。

## 需要釐清的規格

正式實作前要決定：

- 外資連買要幾天
- 投信買超要幾天
- 是否看近 20 日淨買超
- 是否用買超佔成交量比例，而不是只看張數
- 是否加入投信持股比例或籌碼集中度
- 是否只看小型股
- 是否要加流動性門檻
- 是否要排除處置股
- 是否要和月營收 YoY 交叉驗證
- 結果排序依據是什麼

## 初版建議規則

初版可以先用保守規則：

- 投信連續買超 >= 3 日
- 外資近 5 日淨買超為正，作為輔助條件
- 投信買超張數佔當日成交量 > 10%
- 近 20 日日均成交額 >= 500 萬
- 股價在 20MA 之上
- 股價短期漲幅未過熱，避免追高
- 顯示外資 20 日淨買超
- 顯示投信買超天數
- 顯示買超佔成交量比例
- 顯示最近掃描時間

若資料不足，應顯示空結果或錯誤狀態。

不能假裝有訊號。

## 不做

初版不做：

- 自動下單
- 直接產生買進建議
- 複雜機器學習模型
- 分點券商追蹤
- 即時盤中法人資料

## 驗證方式

後端：

- `/api/institution-scan` 可正常回傳
- response 欄位符合前端型別
- 沒有資料時不 crash
- FinMind API 失敗時有合理錯誤處理
- T 日籌碼資料只能用於 T+1 或更晚交易日的回測

前端：

- institution tab 可刷新
- 空結果有明確提示
- 結果點擊後能開股票頁
- 不影響其他 tab

策略：

- 不可標記為已驗證策略
- 若要變成實盤候選，必須另做回測
