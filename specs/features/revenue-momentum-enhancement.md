# Feature Spec: 月營收動能增強

更新日期：2026-04-16

## 目標

增強目前月營收動能掃描策略。

月營收動能是目前最可信的主策略。

後續工作不是推翻它，而是逐步加強訊號品質與風控。

## 目前狀態

已存在：

- `/api/revenue-scan`
- `stock_revenue_monthly`
- 月營收 YoY 計算
- 流動性過濾
- 大盤 200MA 濾網
- 前端 scan tab 顯示結果

已知回測結果：

- Test 2023-2025 年化報酬約 29.1%
- Sharpe 約 1.74
- Max Drawdown 約 -13.3%

這是目前最應保留與增強的策略主線。

## 當前執行優先

第一步：先測 YoY 持續性。

其他方向待此步驟完成後再排序。

## 可增強方向

### 1. YoY 持續性

不只看單月 YoY。

也看：

- 連續幾個月 YoY 為正
- YoY 是否加速
- 最近 3 到 6 個月平均 YoY

初版可回測規則：

- 最新月營收 YoY > 20%
- 最新月營收 MoM > 0
- 最新月營收創近 12 個月新高
- 公告日後下一個可交易日才可進場
- 股價在 20MA 與 60MA 之上
- 排除日均量 < 500 張或日均成交額過低標的

### 2. 營收驚喜

比較本月 YoY 和過去幾個月平均。

目標是找出突然改善，而不是只找已經很熱門的股票。

### 3. 法人籌碼交叉驗證

可和法人籌碼掃描結合：

- 月營收強
- 外資或投信同步買超
- 流動性足夠

這可能提升訊號品質。

但需要回測，不可直接當成真。

### 4. 風控濾網

可加入：

- 大盤 200MA
- 處置股排除
- 流動性門檻
- 漲跌停不可成交
- 產業集中度限制

## 不做

初版增強不做：

- 自動下單
- 複雜機器學習模型
- 大規模 UI 重設計
- 同時改多個策略

## 驗證方式

任何增強都要回測。

至少要比較：

- 原始月營收策略
- 增強後策略
- Train / Test 分段
- 交易次數
- Sharpe
- MaxDD
- 平均單筆報酬
- 扣交易成本後結果
- 公告日 T+1 才進場的版本

若增強後只是讓 Train 變好，但 Test 變差，不算成功。

若回測使用月營收資料但沒有正確處理公告日，也不算成功。

## 完成標準

一次只做一種增強。

例如：

- 先測 YoY 持續性
- 再測營收驚喜
- 再測法人籌碼交叉

每次變更都要有：

- feature spec
- 回測輸出
- 結論
- 文件同步
