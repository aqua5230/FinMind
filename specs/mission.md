# Mission

更新日期：2026-04-16

## 專案目標

FinMind 的目標是建立一套台股量化研究、掃描與驗證系統。

它不是單純展示股價資料。

也不是直接自動下單系統。

目前最重要的任務是：

- 找出台股市場中可被程式化驗證的策略線索
- 用回測確認策略是否真的有正期望值
- 把可用策略做成掃描器
- 用前端量化終端機輔助觀察與決策

## 使用者

主要使用者是專案擁有者本人。

使用情境：

- 研究台股策略
- 檢查回測結果
- 掃描候選股票
- 觀察策略是否值得繼續開發

短期內不假設一般大眾使用。

## 目前產品形態

目前產品比較像：

台股量化研究工作台。

它包含：

- 策略研究
- 回測
- 掃描器
- 前端觀察介面
- 部署中的 API

目前不應把它視為：

- 全自動交易系統
- 財務建議工具
- 已驗證可實盤的策略集合

## 成功標準

第一階段成功，不是找到很多策略。

而是建立一套不會自欺的流程。

成功標準：

- 每個策略都有清楚規格
- 每個策略都有回測結果
- 失敗策略被明確標記或移除
- 可疑結果不被包裝成可實盤訊號
- 前端掃描器只展示目前仍值得追蹤的策略

## 暫時不做

目前暫時不做：

- 自動下單
- 實盤交易
- 對外公開服務
- 複雜會員系統
- 多使用者權限
- 大規模重構

若未來要進入實盤，必須先補：

- 交易成本模型
- 滑價模型
- 漲跌停不可成交規則
- 風控規格
- 手動確認流程
- 交易紀錄與回滾機制

## 核心原則

- 勝率不等於賺錢
- 沒有完整回測，不進實盤
- 策略失敗要留下教訓，不要只刪掉痕跡
- 新策略先寫 feature spec，再寫程式
- 回測標準比單次漂亮結果重要
- AI 可以協助開發，但不能替代人類判斷策略是否可信
